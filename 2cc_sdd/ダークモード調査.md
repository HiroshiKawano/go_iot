## 0. 前提と結論サマリ

- **既存ダークモード実装はゼロ**（`prefers-color-scheme` / `data-theme` / トグルUI の痕跡なし。lib生成物を除く全 grep で確認）。フルスクラッチで追加する。
- **追い風**：色は CSS カスタムプロパティ（`:root` トークン）に集中定義されており、多くの UI は `var(--color-*)` を参照。トークン差し替えで大半が切り替わる。templ 内にインライン色は**ゼロ**（検証済み）。FlashMessage は無スタイル（body 継承）で自動追従。
- **障害は 4 系統**（v2 で 2→4 に増補）：
  1. **CSS 内のトークン化されていない生色**（黄バナー・プレースホルダ縞・十字線・select 矢印 SVG・影）→ §3
  2. **ECharts のクローム**（軸・凡例・ツールチップ・visualMap・dataZoom）が既定色（暗テキスト）依存 → §6
  3. **トークンが届かない領域**：CDN の Tom Select CSS（白背景ハードコード）と、ネイティブ UI（date ピッカー・checkbox・スクロールバー）→ §1.5・§4
  4. **トークン誤用バグ 2 件**：`.error-toast` と `.badge-caution` が「白のつもり」で `var(--color-surface)` / `var(--color-text)` を文字色に使っており、**ダーク値を入れた瞬間にコントラスト崩壊**する → §3 後半（要修正）
- **運用制約**：CSS は**モック単一ソース運用**。正本は `mocks/html/style.css`。本番 CSS は生成物で手編集禁止。変数追加・トグルUI 追加は**モック HTML と正本 CSS の両方に反映**が必須（デグレ防止）。

| `--color-primary` | `#28a745` | 緑・主アクション | 微明化 `#2fbe54` |

| `--color-vpd/dewpoint/gdd/trend/heat` | （§1.4 参照） | **データ意味色**（系列・バッジ共用） | **両テーマ据置**（沈む色のみ個別微調整） |

**2 層トークン設計（外部レビュー採用）**：ダーク値を `@media` と `[data-theme=dark]` に二重記述する重複を避けるため、
① 原色パレット層（`--palette-*`：テーマで変えない生値）と ② セマンティック層（`--color-*`：役割名。テーマで再マップ）に分離する。
ダークブロックはセマンティック層の再宣言 1 箇所のみとなり、保守点が一元化する。

**新設トークン（v2・§3 のバグ修正と対）**：
- `--color-on-accent`: 彩色地（primary/danger 等）の上の文字色。両テーマ `#fff` 固定。
- `--color-on-warning`: warning 地の上の文字色。両テーマ `#212529` 固定（黄色地に明文字は不可）。

> 影トークン `--shadow-sm/md`（`rgba(20,20,21,.05)`）はダーク背景でほぼ不可視。ダークでは影を濃くする（例 `rgba(0,0,0,.4)`）か、境界線でカード分離を表現。

## 4. 実装方式

### 方式比較

| 方式 | 仕組み | 長所 | 短所 |
|---|---|---|---|
| A. OS 追従のみ | `@media (prefers-color-scheme: dark)` | 最小工数・JS 不要・FOUC なし | 手動選択不可 |
| B. 手動トグルのみ | `data-theme` ＋ localStorage | 明示切替 | JS 必須・FOUC 対策要・OS 好み無視 |
| **C. 両立（推奨）** | 既定 OS 追従＋`data-theme` 手動上書き | UX 最良 | 実装やや増 |

### 方式 C の具体設計（v2 更新）

1. **トークン定義（CSS・正本＋モック両方）**：§2 の 2 層化。ダークブロックは
   `@media (prefers-color-scheme: dark) { :root:not([data-theme="light"]) {…} }` と `:root[data-theme="dark"] {…}` の両導線からセマンティック層を再マップ。
2. **`color-scheme` の宣言（v2 追加・ROI 最大）**：ダーク適用時に `color-scheme: dark;` を `:root` へ。
   ネイティブ UI（**date ピッカー・checkbox・radio・スクロールバー・select ポップアップ**）が一括ダーク化する。
   本システムのネイティブコントロール実数（templ+モック grep）：`type=date`×5・`checkbox`×4・`radio`×2・`number`×1・select 多数 → **効果大**。
   併せて `accent-color: var(--color-primary)` を指定すると checkbox/radio がブランド緑になる（両テーマで有効な改善）。
3. **FOUC 対策**：`<head>` の `<link>` **前**に極小インラインスクリプト（`localStorage.theme` → `documentElement.dataset.theme` 即設定）。App/Guest 両方。**CSP 不在確認済みでブロックされない**（§1.3）。
4. **トグル UI（App＋Guest 両方・§9 確定）**：切替ロジックは共通コンポーネント（例 `ThemeToggle.templ`）に括り出し、App は `SiteHeader.templ` の `.user-menu` 内・Guest は独自配置（§1.3）で共有する。Alpine で `localStorage` 永続＋`html[data-theme]` 更新＋**`CustomEvent('themechange')` 発火**（ECharts 連動用。Guest には ECharts が無いため購読側は不在でよく、発火は共通で害なし）。a11y：`aria-pressed`＋`.u-visually-hidden` ラベル。
5. **Tom Select ダーク上書き**：§1.5。正本 CSS に `.ts-dropdown` 系を追加。
6. **ECharts 連動**：§6（v2 で独立章に昇格）。

## 6. ECharts のテーマ対応（v2 全面増補・最大の工数点）

### 6.1 描画構造（前提）

- サーバ側 `internal/chart/*.go`（go-echarts v2.7.2）が option を構築 → `<script type="application/json" id="<id>-option">` に埋込 → クライアントの **EChartsInitializer**（[App.templ:126-212](../internal/view/layout/App.templ#L126-L212)）が parse→init→後処理（endLabel/sampling/tooltip.formatter）→setOption。
- HTMX 期間切替 swap で `htmx:beforeSwap`→dispose / `htmx:afterSwap`→init。

### 6.2 対象コンテナの全数（12 個・v2 実測）

`[data-echarts]` は device-show に 11（`temperature-chart` / `humidity-chart` / `diurnal-chart` / `vpd-chart` / `dewpoint-chart` / `gdd-chart` / `thi-heatmap` / `tropical-night-calendar` / `night-temp-delta` / `ah-line` / `tropical-night-trend`）＋ analysis-trend に 1（`trend-chart`）。**line だけでなく heatmap / calendar / bar を含む**。

### 6.3 現状：クロームは全チャートで既定色依存（ダークで判読不能）

サーバ側は `axisLabel` / `textStyle` / `splitLine` / `backgroundColor` を**一切指定していない**（chart pkg 全 grep で確認）。ダーク背景で問題になる既定クローム：

| クローム | 既定 | ダークでの症状 | 該当チャート |
|---|---|---|---|
| 軸ラベル・軸線 | 暗グレー | ほぼ見えない | 全 12 |
| 凡例テキスト | `#333` 系 | 見えない | 温湿度/夜温ΔT/トレンド等の legend 持ち |
| ツールチップ | 白地・暗文字 | 読める（浮くが機能はする）→ 任意で暗地化 | 全 12 |
| **visualMap**（連続バー・min/max ラベル） | 暗文字 | ラベル判読不能 | `thi-heatmap` / `tropical-night-calendar`（[heatstress_echarts.go:52-62](../internal/chart/heatstress_echarts.go#L52-L62) に textStyle なしを確認） |
| **dataZoom スライダー** | 明地クローム | 白帯が浮く | `trend-chart`（[trend_echarts.go:52-55](../internal/chart/trend_echarts.go#L52-L55) で slider 使用を確認） |
| calendar の枠線・月ラベル | 明地前提 | 要目視 | `tropical-night-calendar` |

一方、**据置でよいもの**：系列色（データ意味色）、endLabel（系列色を継承）、markPoint max/min（彩色ピンに白文字）。

### 6.4 実装方式：クライアント側「クローム patch」（テーマ theme 機能は使わない）

- **サーバ option はテーマ非依存の中立を維持**（系列色のみ保持）。クロームの着色はクライアントで行う。
- `echarts.init(el, 'dark')` の組込テーマや `registerTheme` は**不使用**。理由は「patch 方式のほうが変更面が最小で予測可能」だから（※系列の明示色は theme に優先されるので theme でも上書きはされないが、挙動面を広げない設計判断）。
- **patch は 2 タイミングで適用**（v2 で明確化・片方だけだと漏れる）：
  1. **initContainer 時**：`setOption(option)` の前に現在テーマのクローム色をマージ。**ダーク中に HTMX 期間切替で再生成されるグラフ**もこれで正しく着色される。
  2. **`themechange` イベント時**：既存全インスタンスへ `setOption(patch)` を再適用。**dispose→init はしない**（dataZoom のズーム位置・凡例トグル状態が飛ぶため。setOption マージなら保持される）。
- **軸配列の罠（外部レビュー指摘・裏取り済み）**：乖離率オーバーレイ時のみ [echarts.go:89](../internal/chart/echarts.go#L89) の `ExtendYAxis` で **yAxis が配列（2 本）**になる。patch は固定形でなく**既存 option の xAxis/yAxis の本数を読んで同数分生成**する汎用ヘルパにする（heatmap の category 軸・calendar の無軸もこれで吸収）。
- `tooltip.formatter` は初期化スクリプトが既に上書きしている（[App.templ:166-180](../internal/view/layout/App.templ#L166-L180)）ため、patch では formatter を触らず色プロパティのみをマージする。

### 6.5 markArea 注入色（半透明ウォッシュ）の扱い【v2 新規】

サーバ側 Go 定数として注入される半透明帯が 6 種ある（全列挙・裏取り済み）：

| 定数 | 値 | 用途 |
|---|---|---|
| `vpdZoneDryColor` | `rgba(230,126,34,0.10)` | VPD 乾きすぎ帯 |
| `vpdZoneOptimalColor` | `rgba(64,160,90,0.12)` | VPD 適正帯 |
| `vpdZoneWetColor` | `rgba(52,120,200,0.10)` | VPD 湿りすぎ帯 |
| `condensationBandColor` | `rgba(66,99,235,0.12)` | 結露帯 |
| `gapBandColor` | `rgba(120,120,120,0.12)` | 連続欠測帯 |
| `trendSignificantBandColor` | `rgba(112,72,232,0.10)` | 有意区間帯 |

10〜12% の色ウォッシュは暗地では視認性が落ちる可能性がある。**方針：まず据置で実機スモーク → 不足なら「両テーマで成立する単一の不透明度」に微調整**（テーマ別値にするとサーバ option の中立性が崩れるため最終手段）。正常帯（`AreaStyle Opacity 0.15`・系列色）も同様。

## 8. テスト観点

- **templ レンダリング**（トグル要素・FOUC スクリプト・`data-theme` 属性）は既存 component/handler テスト方式で検証可能。
- ECharts クローム patch はクライアント JS のため Go テスト外 → E2E（Playwright）でダーク時の軸ラベル可視性・テーマ切替後の dataZoom 状態保持を確認する余地。
- **CSS のトークン誤用**（§3 のバグ類型）は grep で機械検査できる：「`color: var(--color-surface|--color-text)` が彩色 background と同居する宣言」を禁止パターン化。
- 意味色の向き（暖=暑/乾、寒=湿）はダークでも不変。**実機スモーク必須**（VPD 規約の過去事例）。
