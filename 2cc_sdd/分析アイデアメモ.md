# 農業IoT データ分析アイデアメモ（温度・湿度）

> **ステータス**: アイデア段階（未実装・未仕様化）
> **作成日**: 2026-06-25（同日、実装ロードマップ形式へ再構成）
> **対象データ**: 各センサーボードが収集する **温度・湿度** の時系列ログ（まずはこの2変数から）
> **目的**: 蓄積した温湿度データから「誰が・何を・どの統計手法で分析するか」を、**上から順に実装できるロードマップ**として整理する。末尾に数式リファレンスを付ける。

---

## 0. このメモの読み方（構成）

当初は「統計手法カタログ（A〜F）」と「眞境名さんのニーズ（旧G章）」が別々に並んでいたが、**そのままでは上から実装できない**ため、以下の3層に組み直した。

```
1. 顧客像と方針の核     … 誰のための分析か（=実装の優先順位を規定する）
2. 実装ロードマップ      … 上から順に実装。各フェーズ＝「ニーズ(確度)・作るもの・使う手法・主要数式」
   （フェーズ0 → 7）       ← ★ここが背骨。これを上から潰せばよい
3. 付録: 数式リファレンス … A〜F の手法・指標カタログ（ロードマップから参照される）
```

- **確度の凡例**: 〔明示〕＝引継ぎメモに記述あり ／ 〔強く示唆〕＝構成から読み取れる ／ 〔推測〕＝踏み込んだ推測 ／ 〔将来〕＝後続フェーズ。
- ロードマップ各フェーズの「使う手法」は **付録 A〜F / D①〜⑥** を指す。数式の本体は付録にある。

## 1. 顧客像と方針の核

実装の優先順位は「誰のための分析か」で決まる。ここを最初に固定する。

### 1-1. 最重要の再フレーム：眞境名さんは「農家」ではなく「研究者・公務員」

眞境名 元次さん＝沖縄県農業研究センター 農業システム開発班 主任研究員。商業生産者ではなく**研究・普及側**。同じ温湿度データでも「分析の目的関数」が農家とは根本的に違い、これがロードマップ全体の並びを規定する。

| 観点 | 商業農家 | 眞境名さん（研究・普及） |
|---|---|---|
| **目的** | 自分のハウスの今を見て今日の操作（換気/灌水）を決める＝単一圃場のリアルタイム意思決定 | 複数条件を突き合わせ**再現性あるエビデンス**を作り県内農家へ普及指導＝条件間比較とエビデンス生成 |
| **時間軸** | 直近〜数日 | **多シーズン・多年の長期蓄積**で平年値化・作型比較・季節性分解 ＝ データ主権（自前永続保存）が必須動機 |
| **データの扱い** | アプリ内で完結 | **CSVで外部（R/Excel/python）へ持ち出し**、回帰・分散分析・SARIMA を自分の手法で何度も再解析し、論文・報告書・研究会発表の図表に再現可能な形で使う |
| **品質要件** | ざっくりで可 | 査読・報告書品質のため**欠測率・センサ故障・器差・品質フラグの明示が必須**。「どの期間が信頼できるか」を示せないデータは使えない |
| **指標の抽象度** | 生のRHでも実用 | **「生RHを眺めるのは素人。VPD/GDD/露点で語りたい」**＝植物生理に基づく派生指標への変換が前提 |
| **横展開責任** | 自分の利益 | 県全域の作物・圃場をまたぐベンチマークと、研究成果を現場が使える指導ツールへ翻訳する**普及責任** |

> **結論**: 眞境名さんの未充足ニーズは、現状画面（直近24hグラフ＋平均/最高/最低）の**先**＝**多地点比較・派生指標・長期蓄積・CSV外出し・品質保証**にある。Ambient脱却の核心は「保持期間とエクスポート制約が研究の致命傷」だから。
> **由来**: 引継ぎメモ（`2cc_sdd/農業IoT（眞境名さん案件）引継ぎメモ.md`）と本メモを突き合わせ、5視点（研究者本人／施設園芸環境制御／時系列統計／沖縄亜熱帯農業／農業普及行政）から推測・統合（2026-06-25）。

### 1-2. 基本方針：生の温湿度より「派生指標」

**「生の温湿度をそのまま統計にかけるより、農業的に意味のある"派生指標"に変換してから分析する方が圧倒的に効く」**。プロの施設園芸は生の相対湿度をほぼ見ず、**飽差（VPD）** や **積算温度（GDD）** で環境を管理する。手法は付録に以下の6カテゴリで整理してある。

```
生データ(温度・湿度)
   ├─→ A. 基本統計量            … 土台
   ├─→ B. 時系列・移動平均系     … トレンド/乖離
   ├─→ C. 温度×湿度の関係       … 同期率・乖離率
   ├─→ D. 農業特化の派生指標     … ★本命(飽差・積算温度・露点 等)
   ├─→ E. 異常検知              … 警告・品質
   └─→ F. 予測                  … 将来フェーズ
```

眞境名さん当初案（平均気温・移動平均線・移動平均線との乖離・温湿度の同期率/乖離率）は A / B / C に対応し、物理的に最も意味のある「乖離」は D の **飽差（VPD）** に集約される。

### 1-3. ゴールで重み付けは変わる

下のロードマップは「研究者ニーズ」を既定の優先順とした標準形。最終ゴール次第で前倒しすべきフェーズが変わる。

- **(a) リアルタイム警告**（異常・結露・高温ストレス）→ フェーズ3（露点/病害）・5（THI）を前倒し
- **(b) 生育/収穫予測** → フェーズ4（GDD）を前倒し
- **(c) 環境最適化**（飽差を適正帯に）→ フェーズ3（VPD）を前倒し
- **(d) まずは見える化** → フェーズ1で一旦止めて運用

## 2. 実装ロードマップ（上から順に実装）

依存関係と確度で並べた。**各フェーズは「狙い／作るもの／使う手法（付録参照）／主要数式」**で自己完結している。フェーズ0→7を上から潰していけばよい。

---

### フェーズ0：データ基盤と品質保証（全分析の土台）〔明示〕

> 対応ニーズ: 生データ主権・CSV/全期間エクスポート（最優先①）＋ 研究データ品質保証 ＋ 集計軸/多項目拡張の先行設計

**狙い**: 全分析が乗る土台。生データと派生指標を欠落なく自前DBへ永続保存し、CSVで外部へ出せるようにする。同時に**集計軸とスキーマを最初に固定**する（後付けはスキーマ破綻を招くため）。

**作るもの（機能/画面/出力）**
- 生データ＋派生指標列を保存するスキーマ（VPD/露点/GDD/THI/日較差/絶対湿度 を集計列に持てる設計）
- **任意期間×地点×項目フィルタの CSV ダウンロード**（将来は Parquet 等オープンフォーマット）
- 圃場/地点/期間別の日次・時間別集計表（平均/最高最低/日較差/適正帯滞在率/結露時間）を帳票化
- **集計軸（圃場×地点×作物×期間×指標）を最初から固定**
- 多項目（CO2/照度/土壌水分/電圧）追加後も同一パイプで吐ける拡張前提の列設計
- **品質メタ層**: レコード単位の品質フラグ（欠測/stuck/flatline/物理異常/外れ値）、欠測率・サンプリング間隔一貫性・通信遅延の監視、同一ハウス複数センサ間の差分＝器差（キャリブレーションずれ）検証
- 長期蓄積を前提とした永続保存（保持期間を自前で制御）

**根拠**: Ambient脱却の最大動機＝データ主権（引継ぎメモに繰り返し明示）。研究は再現性・査読品質が前提で、品質フラグなしのデータは論文・報告書に使えない。塩害＝機器劣化・多台数運用でドリフト/故障リスクが高い沖縄では品質メタ層が特に効く。現状画面の「通信遅延」列はその芽。

**使う手法 → 付録**: E（Zスコア・IQR・stuck/flatline・物理範囲チェック）／ A（欠測率/分散/通信遅延の集計）／ CSVは A〜F 全分析の入力層。本格統計（回帰/分散分析/SARIMA）はアプリに閉じ込めず外部ツールへ出す前提。

### フェーズ1：見える化の最短形（当初案・最小ゴール）〔明示〕

> 対応ニーズ: 日常監視ダッシュボード（移動平均線・乖離率・正常帯）＋ 日次基本統計

**狙い**: 引継ぎメモの最小ゴール＝「直近24時間の温湿度グラフ」。眞境名さん当初案（平均気温・移動平均線・移動平均線との乖離）をそのまま形にする、見える化の入口。後段の VPD/GDD 分析の前提基盤。

**作るもの**
- 直近24時間の温湿度グラフ
- SMA/EMA/WMA 移動平均線の重ね描き
- 移動平均**乖離率（%）**の表示
- **SMA±kσ のボリンジャー的「正常帯」バンド**（いつもの昼夜変動からの逸脱を視覚的に拾う）
- 現在値/最高/最低/**日較差**の数値カード
- 日次の 平均/最高/最低/日較差/標準偏差/変動係数CV 集計

**使う手法 → 付録**: B（SMA/EMA/WMA・乖離率%・バンド SMA±kσ）／ A（平均・標準偏差・日較差ΔT・CV）

**主要数式**: `SMA = (1/N)Σx` ／ `EMA: α=2/(N+1)` ／ `乖離率 = (x−SMA)/SMA×100` ／ `日較差 ΔT = Tmax − Tmin`

---

### フェーズ2：多地点比較（研究の本丸）〔明示〕

> 対応ニーズ: ハウス内外・東西の多地点比較と空間ムラの定量化（最優先②）
> 前提: フェーズ0で「内/外」「東/西」を地点属性として構造化済みであること。

**狙い**: 「ハウス内外温湿度」「温湿度_東/西」を同一時間軸で突き合わせ差分を定量化。Ambientはチャンネル単位で孤立し地点間を統計的に突き合わせられない＝これが自前化の筆頭差別化点であり、研究＝条件間比較の本丸。沖縄は強日射ゆえ片側偏熱（東西差）が本土より大きく出やすい。

**作るもの**
- 複数センサーを束ねた地点間比較ダッシュボード（同一時間軸の重ね描き）
- **差分系列 ΔT（内-外）／ ΔRH（東-西）** の時系列グラフ
  - (a) 夜間の正の差＝**保温効果** ／ (b) 日中の過剰な正の差＝**オーバーヒート（換気不足の昇温）** ／ (c) 換気稼働時間帯に差が縮小＝**換気効果** ／ (d) 午前は東が高温（朝日）/午後は西が高温（西日）＝**日内ムラ** を、平均差・最大差・差の開く時間帯として定量化
- 内外・東西それぞれの **VPD 比較ビュー**（どちらが乾きすぎ/湿りすぎを1指標で）
- 地点間バンド幅（最大-最小）による**不均一度スコア**（将来の多点化前提）

**使う手法 → 付録**: A（地点別の平均/最高最低/日較差/標準偏差でばらつき）／ C（2地点を温度×温度・湿度×湿度で相関/共分散＝同期度）／ D①（地点別VPDの差）／ B（地点別SMA/EMA重ね描き・差分＝系列演算）

---

### フェーズ3：派生指標で「農業に効く」へ（差別化の核）〔強く示唆〕

> 対応ニーズ: 飽差(VPD)の適正帯評価（最優先③）＋ 露点・葉面湿潤による病害リスク

**狙い**: 生RHではなく植物生理を支配する **VPD** で環境を語る＝Ambientにない値。ここで「ただのグラフ」から「農業に効くダッシュボード」へ跳ねる。さらに露点・葉面湿潤時間で病害リスクを蓄積解析する。本人が「生RHは素人、VPDで語りたい」と明言。

**作るもの（VPD）**
- VPD 時系列グラフ（**適正帯バンドの塗り分け**：適正/乾きすぎ/湿りすぎ、多くの作物で 0.3〜1.5 kPa）
- **適正帯滞在率（日次%）**＝「1日のうち何%が適正帯にあったか」（農家に説明しやすく改善効果を定量化しやすい）
- 時間帯別の VPD 逸脱（日中高VPD＝蒸散過多→気孔閉鎖→しおれ ／ 夜間・スコール時の低VPD＝過湿→結露・病害）
- 作物選択に連動した適正帯しきい値の切替
- VPD の移動平均トレンド

**作るもの（露点・病害）**
- 露点温度の時系列（気温との接近＝**結露帯**のマーク）
- **葉面湿潤時間（高湿度継続時間）の日次積算** → 病害リスクスコア
- 高湿度継続イベントの抽出と一覧
- **発病記録と環境の事後突合用データ蓄積**（「どの温湿度パターンの後に発病したか」を遡れる形で）

**根拠**: 沖縄は高温多湿で昼にVPD過大/夜に飽差不足の両極が起きやすい。梅雨・台風・スコールで葉面湿潤時間が伸び病害圧が高く、病害予察は研究センターの本務。

**使う手法 → 付録**: D①（飽差VPD）／ D③（露点Td）／ D⑤（葉面湿潤→病害スコア）／ A（適正帯滞在率・高湿度連続時間の集計）／ E（適正帯逸脱・高湿度継続のイベント検出）

**主要数式**: `es(T)=0.6108·exp(17.27T/(T+237.3))` ／ `VPD=es·(1−RH/100)` ／ 露点 `Td=237.3γ/(17.27−γ), γ=ln(RH/100)+17.27T/(T+237.3)`

### フェーズ4：蓄積を生かす（季節性・生育予測・環境応答）〔明示／強く示唆〕

> 対応ニーズ: 長期トレンド・季節性 ＋ 積算温度(GDD)で生育/収穫予測 ＋ 温湿度相関で環境応答特性化

**狙い**: 長期蓄積（データ主権）の本領。多年・多シーズン横断で季節性・平年比を論じ、GDDで生育ステージ・収穫期を予測し、相関で施設の挙動を物理的に理解する。

**作るもの（季節・長期トレンド）**
- 日次→週次→月次→年次へロールアップした統計サマリ
- 日較差ΔTの日次推移グラフ
- **ACF** による日周リズム健全性チェック（ラグ24h）
- **線形回帰／Mann-Kendall** による長期トレンド判定
- **STL分解**（トレンド+日周変動+残差）ビュー
- 平年値との比較（今季 vs 平年。「今季は平年比で何℃高い」「梅雨入り前後でどう変わるか」）

**作るもの（積算温度 GDD）**
- 作物選択UI（Tbase連動）
- 定植/播種日からの **GDD累積曲線**
- 収穫適期までの残り積算温度の表示
- 線形回帰による到達日予測の外挿
- 作型比較（作期をずらした時の到達日数差）を地点/圃場別に
- 「沖縄の気温条件での生育ステージ⇔GDD対応表」（作付計画・出荷予測・栽培暦の科学的更新）

**作るもの（温湿度相関・環境応答）**
- 地点別/内外別の T-RH ピアソン相関（通常は負相関）
- **ラグ付き相互相関**（湿度が温度に何分遅れて追従するか）
- 回帰 y=ax+b による環境応答特性（熱容量・換気の効き・気密性能）の定量化
- 換気・灌水後の応答速度の内外/東西比較（構造的ムラの原因究明・改善提案の根拠）

**根拠**: 沖縄は冬も温暖でGDDが速く積算し多期作・周年栽培が可能＝GDDの実利が大きい。相関は「出発点（本命はVPD）」なので優先度は中位だが、施設構造の理解に効く。

**使う手法 → 付録**: A（日平均/Tmax/Tmin/CV）／ B（SMA/EMA・STL・ACF・線形回帰/Mann-Kendall・累積和）／ C（相関・ラグ付き相互相関・回帰）／ D②（GDD）／ F（到達日予測の発展）

**主要数式**: `GDD=Σ max((Tmax+Tmin)/2 − Tbase, 0)` ／ ピアソン `r = Σ(x−x̄)(y−ȳ)/√(Σ(x−x̄)²Σ(y−ȳ)²)`

---

### フェーズ5：沖縄特化の踏み込み（イベント・複合ストレス）〔推測〕

> 対応ニーズ: 台風・スコールのイベントスタディ ＋ 高温ストレス(THI)・熱帯夜・絶対湿度

**狙い**: 沖縄固有の非定常・複合ストレスを扱う。台風常襲ゆえ「非定常イベントが環境にどう響き何時間で復旧したか」の事後解析、熱帯夜ゆえ本土の教科書値が通らない品質低下要因の定量化。

**作るもの（台風・スコール イベントスタディ）**
- 変化率（微分）による急変点の自動マーク
- イベント前後の比較ビュー（気温急降下・湿度急上昇・極値・回復に要した時間）
- **イベント台帳**（台風号数・日時・影響度。防災対応・被覆資材評価・災害報告の根拠）
- STL残差ベースの異常抽出

**作るもの（THI・熱帯夜・絶対湿度）**
- THI 高温ストレス帯の時間帯別集計
- **熱帯夜（夜温25℃以上）の連続日数**・夜間最低気温の推移・夜温が下がらない期間
- 絶対湿度AHによる除湿負荷の可視化
- 日較差ΔT（糖度・着果に直結）と品質の関係づけビュー（「夜温が下がらず日較差が稼げない→糖度低下」）

**根拠**: 沖縄＝台風常襲・スコール多発で本土よりイベントスタディの必要性が格段に高い。熱帯夜が長く日較差が稼ぎにくいのは沖縄固有の品質低下要因で、品種選定・遮光指導のエビデンスになる。

**使う手法 → 付録**: B（変化率＝微分・STL残差）／ E（急変検出・閾値超過）／ A（イベント前後の極値/回復時間・連続日数の季節集計）／ D⑥（THI）／ D④（絶対湿度AH）

**主要数式**: `変化率 Δ=(x_t−x_{t-1})/Δt` ／ `THI=0.8T+(RH/100)(T−14.4)+46.4`

## 3. 横断的な設計上の注意（フェーズをまたぐ盲点）

個別フェーズの機能に目が行きがちだが、以下は**複数フェーズにまたがる設計判断**で、最初に決めておかないと後で破綻する。

- **① 集計軸を最初に固定**（圃場×地点×作物×期間×指標）。研究者の真の需要は「どの軸でも横並び集計・CSV化できる」こと。後付けで軸を足すとスキーマ破綻。多項目追加（フェーズ6）とセットで先に決める。→ フェーズ0
- **② データ品質メタ層を最初から**（欠測率・stuck/flatline・物理異常・器差・品質フラグ）。研究者は「そのデータが信頼できるか」を担保できないと論文・報告書に使えない。地味だが必須で見落とされやすい。→ フェーズ0
- **③ 研究用画面と農家共有画面を分離**。高度指標（VPD/GDD/病害スコア）をそのまま出すと現場で使えない。信号色・適正帯バンド・平易ラベルへ落とす**翻訳レイヤー**が普及側ならではの隠れ要求。→ フェーズ6
- **④「時間割合（滞在率）」の発想**。瞬間値や平均でなく「1日のうち何%が適正帯にあったか」が農家に説明しやすく改善効果を定量化しやすい。逸脱の量と時間帯をセットで集計する設計。→ フェーズ3
- **⑤ 作物メタデータの保持**。GDDのTbase・VPD適正帯・病害モデルは作物依存。作物マスタで切り替えられる設計。→ フェーズ3・4・6

# 付録. 数式・指標リファレンス（A〜F カタログ）

ロードマップ各フェーズの「使う手法」が参照する、手法・指標の数式集。

## A. 基本統計量（土台・すぐできる）

| 指標 | 数式 | 農業的な意味 |
|---|---|---|
| 平均 μ | `μ = (1/N)Σxᵢ` | 期間の代表値 |
| 中央値 / 最頻値 | 順序統計量 | 外れ値に強い代表値 |
| 最高 / 最低 | `T_max`, `T_min` | 日内の極値。熱・冷害の把握 |
| 標準偏差 σ | `σ = √((1/N)Σ(xᵢ-μ)²)` | 環境の安定度。大きい＝乱高下 |
| 分散 σ² | `σ² = (1/N)Σ(xᵢ-μ)²` | ばらつきの基礎量 |
| 変動係数 CV | `CV = σ/μ` | 平均で正規化したばらつき。温度帯が違う日同士を比較できる |
| 日較差 ΔT | `ΔT = T_max - T_min`（その日） | **果実の糖度・着果に直結**。農業では平均より重要なことが多い |
| 範囲 / 四分位 / IQR | `IQR = Q3 - Q1` | 外れ値に強いばらつき。異常検知の土台 |

**ポイント**: 日較差は地味だが、トマトの糖度や花芽分化に効く実用指標。最初から入れる価値がある。

---

## B. 時系列・移動平均系

```
単純移動平均 SMA  : SMA_t = (1/N) Σ_{i=t-N+1}^{t} xᵢ
加重移動平均 WMA  : WMA_t = (Σ wᵢ·xᵢ) / (Σ wᵢ)        ← 直近ほど重み大
指数移動平均 EMA  : EMA_t = α·x_t + (1-α)·EMA_{t-1},  α = 2/(N+1)
                   ← 直近を重視。反応が速い
乖離率(disparity) : 乖離率 = (x_t - SMA_t) / SMA_t × 100  [%]
変化率(微分)      : Δ = (x_t - x_{t-1}) / Δt   ← 急変検出（ドア開放・機器停止）
```

「移動平均線との乖離」は株のテクニカルでいう **乖離率** そのもの。さらに発展させると：

- **ボリンジャーバンド的アプローチ**: `SMA ± k·σ`（移動窓のσ）で「正常帯」を作り、外れたら警告。乖離率を統計的にしきい値化できる。
- **STL分解**: 時系列を `トレンド + 周期(日周変動) + 残差` に分解。残差だけ見れば「いつもの昼夜変動を除いた異常」が拾える。
- **自己相関 ACF**: ラグ24hで強い相関 → 健全な日周リズム。それが崩れたら異常。
- **トレンド検出**: 線形回帰の傾き、または Mann-Kendall 検定で季節をまたぐ長期傾向を判定。

---

## C. 温度 × 湿度の関係

```
ピアソン相関  : r = Σ(xᵢ-x̄)(yᵢ-ȳ) / √(Σ(xᵢ-x̄)²·Σ(yᵢ-ȳ)²)
               ← 温度と相対湿度は通常「負の相関」(昼:温度↑→RH↓)
相互相関(ラグ) : 湿度が温度に何分遅れて追従するかを検出
共分散        : cov(x,y) = (1/N)Σ(xᵢ-x̄)(yᵢ-ȳ)
回帰          : y = a·x + b で関係を定量化
```

**重要な注意**: 「温湿度の同期/乖離」を相関係数で測るのは出発点としては良いが、物理的に最も意味のある"乖離"は **飽差（VPD）**。相対湿度は温度に依存して動くため、「温度40℃・湿度50%」と「温度20℃・湿度50%」は植物にとって全く別物。これを1つの数字に統合したのが D（特に①VPD）。

---

## D. 農業特化の派生指標（★本命）

温度と湿度の **2つだけ** から計算できて、生データより遥かに実用的。

### ① 飽差 VPD（最重要）

```
飽和水蒸気圧 es(T) = 0.6108 · exp(17.27·T/(T+237.3))   [kPa]   (Tetens式)
実水蒸気圧   ea     = es(T) · RH/100
飽差 VPD            = es(T) − ea = es(T)·(1 − RH/100)   [kPa]
```

植物の蒸散・気孔の開閉を支配。**施設園芸の環境制御は事実上これを目標値に管理**する（多くの作物で 0.3〜1.5 kPa が適正帯）。「温湿度の真の乖離率」はこれ。

### ② 積算温度 GDD（生育・収穫予測の王道）

```
GDD = Σ max( (T_max + T_min)/2 − T_base , 0 )
      T_base は作物固有（例: 多くの作物で 10℃）
```

「あと何℃積算すれば収穫適期か」を予測できる。**収穫日予測・生育ステージ判定**に直結。

### ③ 露点温度・結露リスク

```
γ  = ln(RH/100) + 17.27·T/(T+237.3)
露点 Td = 237.3·γ / (17.27 − γ)   [℃]
結露リスク: 葉面温度 ≦ Td のとき発生
```

結露 → 灰色かび病・うどんこ病の温床。**病害アラートの基礎**。

### ④ 絶対湿度（容積絶対湿度）

```
絶対湿度 AH ≈ 217 · ea / (T + 273.15)   [g/m³]
            （ea は ①で算出した実水蒸気圧 [kPa] を Pa 換算して使う）
```

相対湿度と違い温度に依存しない「実際の水分量」。換気・除湿制御の判断に有用。

### ⑤ 病害リスクモデル

特定の「温度帯 × 葉面湿潤時間」で発病確率が上がる作物が多い。温湿度ログから **葉面湿潤時間（高湿度継続時間）** を積算してリスクスコア化できる。

### ⑥ THI（温湿度指数 / 高温ストレス）

```
THI = 0.8·T + (RH/100)·(T − 14.4) + 46.4
```

高温多湿の複合ストレスを1指標化。

## E. 異常検知

```
Zスコア      : z = (x − μ)/σ          |z| > 3 で外れ値
ローリングσ法 : 移動窓の μ±kσ を外れたら警告（B のバンドと同じ発想）
IQR法        : x < Q1 − 1.5·IQR  または  x > Q3 + 1.5·IQR
急変検出      : 変化率（微分）が閾値超え → ドア開放・機器停止の疑い
センサー故障  : 同一値がN回連続（stuck/flatline）／ 物理的にあり得ない値
```
