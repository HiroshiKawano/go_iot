## 農業IoTシステム 構成概要（HTMX + Blade 構成）

### **[デバイス層]：ESP8266 + SHT31 (高精度温湿度センサー)**

* **実装**: Arduinoコード (C++)。SHT31からI2Cで値を取得。
* **通信 (Device → Server)**: **REST API (HTTP POST/GET)**
* ※デバイスはブラウザではないため、ここだけは引き続きREST APIを使用します。


* **認証**: ヘッダーに **Sanctum APIトークン** を付与。

---

### **[バックエンド]：Laravel 12 (Blade + HTMX モノリス構成)**

* **役割**: データの受信・保存、SVGグラフのサーバーサイド生成、Bladeへのデータ供給。
* **グラフ生成**: **goat1000/svggraph**（PHPネイティブSVGグラフ生成）
* **ドキュメント**: **Scalar (Device API専用)**
* ※フロントエンド用のAPI定義は不要（BladeがHTMLを直接返すため）、デバイスが叩くエンドポイントのみをドキュメント化します。


* **通信 (Server → Frontend)**: **Bladeレンダリング済みHTML（SVG埋め込み）**
* ※APIエンドポイントを作らず、ControllerがBladeテンプレートを直接レンダリングします。


* **認証**: **Session (Web標準)**
* ※通常のログイン機能（Laravel Breeze）をそのまま利用。



---

### **[フロントエンド]：Laravel Blade + HTMX + Alpine.js**

* **構成**: JavaScriptビルドツール不要。フロント担当者はHTMLモック作成のみ。Alpine.jsでUIの開閉・タブ切替等の軽量インタラクションを実現。
* **可視化**: **goat1000/svggraph**（サーバーサイドSVG生成 → HTML埋め込み）
* ※LaravelサーバーがレンダリングしたSVGをそのままBladeテンプレートに埋め込みます。


* **制御**: HTMXの `hx-get` / `hx-post` / `hx-put` / `hx-delete` / `hx-patch` を使用し、ボタン操作でデバイス設定などを更新。
* **部分更新**: Blade Fragment（`@fragment` / `@endfragment`）を使用。HTMLモックをそのまま1つのBladeファイルに貼り付け、部分更新する箇所を `@fragment` で囲む。部分テンプレートファイルは作成しない。Controller は `->fragment()` で該当範囲のみを返す。
