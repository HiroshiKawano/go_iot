- [x] 3. Core: 表示整形ロジック（handler 内 helper）
- [x] 3.1 未対応アラートの表示文言を合成する
  - 「{デバイス名}: {指標}が{閾値}{単位}{を超えました／を下回りました}（{実測値}{単位}）」を組み立てる。指標・単位はドメインの表示メソッドを使い、比較演算子 `>`/`>=` は「を超えました」、`<`/`<=` は「を下回りました」に写像する。
  - 閾値は末尾ゼロを除去（35.00→35・35.50→35.5）、実測値は小数2桁固定で整形する。
  - 単体テストを先に書く（温度×超過・湿度×下回り・演算子4種・閾値桁の各ケース）。
  - 観測可能な完了: 単体テストが緑で、モック文言（例「ハウスA温湿度計: 温度が35℃を超えました（38.50℃）」）と一致する文字列を返す。
  - _Requirements: 4.2, 4.4_
  - _Boundary: dashboard.go（composeAlertMessage）_

- [x] 3.2 デバイス行を表示用データへ整形する
  - リポジトリのデバイス行＋最新計測（無い場合あり）＋基準時刻から、表示用デバイスデータを生成する。温度・湿度は小数2桁＋単位、最新計測が無い場合は温度・湿度とも「ー」。
  - 最終通信は、通信実績がない（日時が未設定）場合「通信実績なし」、ある場合は相対時間フォーマッタの結果を用いる。
  - 単体テストを先に書く（計測あり/未受信→「ー」・通信実績あり/なし→「通信実績なし」・稼働フラグ）。
  - 観測可能な完了: 単体テストが緑で、欠損ケースを含めて期待どおりの表示用データへ写像される。
  - _Requirements: 2.2, 3.1, 3.2, 3.3, 3.4_
  - _Boundary: dashboard.go（デバイス整形 helper）_
  - _Depends: 1.1, 2.1_
