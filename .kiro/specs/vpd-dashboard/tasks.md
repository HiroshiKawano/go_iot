# Implementation Plan — vpd-dashboard

> 逐次実装（上から1行ずつ `/tdd` の RED→GREEN→REFACTOR で完了）。`(P)` 並列マーカーは付けない。
> 生成依存順を厳守: goose migration 定義 → `make db-snapshot` → `make sqlc` → `make templ` → 生成物を消費する実装。
> 詳細・契約・型は [design.md](./design.md)、要件は [requirements.md](./requirements.md) を参照。

## 1. Foundation — 作物マスタとスキーマ/コード生成

- [x] 1.1 作物マスタ（domain.Crop）と作物別 VPD 適正帯
  - 9作物（ゴーヤ/インゲン/サトウキビ/マンゴー/パイナップル/ウリ/米/いも/葉野菜）の Enum と日本語表示名・有効判定・全列挙・文字列パースを定義する（`metric.go` を写経・`fmt` のみ依存）
  - 作物別 VPD 適正帯（下限/上限 kPa）を返し、未選択（空）・未知・不正作物は既定 0.3〜1.5 kPa にフォールバックする
  - 暫定適正帯は design.md の表どおり（施設果菜=狭め・露地=既定）。値は1メソッド（表）で集約しユーザー/文献確定時に更新可能にする
  - 観測可能完了: 全9作物の表示名・有効判定・適正帯、未設定/不正で既定 (0.3,1.5) を返すユニットテストが緑
  - _Requirements: 2.1, 2.2, 2.3, 2.4, 2.5_

- [x] 1.2 devices.crop 列の追加（goose 00009）と DB スナップショット再生成
  - `devices` に作物列（nullable VARCHAR・FK なし・索引なし）を追加し、許容値を作物マスタの9値に限定する CHECK 制約（NULL 許容）と日本語 COMMENT を付す。Down で列 DROP
  - backfill（DML）は含めない（既定フォールバックで既存デバイスは無改修で動作）
  - `make db-snapshot` を実行し `docs/database_snapshot/` を再生成する
  - 観測可能完了: `make migrate-up`／`down` が成功し、スナップショットに作物列・CHECK 制約が反映される
  - _Requirements: 2.5, 2.6_

- [x] 1.3 デバイス作成/更新クエリへの作物列追加（sqlc 再生成）
  - デバイス作成・更新クエリに作物列を加え（locality と同型・空文字→NULL）、`make sqlc` で `repository` を再生成する
  - backfill 専用の単列更新クエリは追加しない
  - 観測可能完了: 再生成された生成コードがデバイス作成/更新の引数に作物を持ち、ビルドが通る
  - _Requirements: 2.6, 3.2_

## 2. Foundation — VPD 計算・描画の純粋/描画層

- [x] 2.1 VPD 算出・適正帯滞在率の純関数
  - 温度・湿度から飽和水蒸気圧（Tetens 確定定数）経由で VPD [kPa] を算出する関数と、同長スライスから VPD 系列を返す関数を実装する（`math` のみ依存・time/DB/gin 非依存）
  - 湿度100%→0kPa・湿度0%→最大飽差・氷点下で NaN/無限大を出さない・欠測（不一致長）を防御的に扱う
  - 値が適正帯 [下限,上限]（両端含む）に入る割合（0〜1・空入力は0）を返す滞在率関数を実装する
  - 観測可能完了: 既知の手計算値一致・RH 0/100・氷点下・滞在率境界（全在帯/全逸脱/空）のユニットテストが緑
  - _Requirements: 1.1, 1.2, 1.3, 1.4, 1.5, 1.6, 1.7, 5.1, 5.2_

- [x] 2.2 VPD グラフ option ビルダー（適正帯3ゾーン markArea・移動平均）
  - VPD 実測線を主役に、適正帯を3ゾーン（下限未満=乾きすぎ／適正／上限超=湿りすぎ）の塗り分けとして重ねた ECharts option を HTML 安全 JSON で構築する
  - 塗り分けは正しい `yAxis` キーで自前構築する（go-echarts の markArea 型は JSON タグ非準拠＝design D-1 のため使用しない）。y 軸は下限0・上限はパラメータで受け取り3ゾーンを常時可視にする
  - 適正帯しきい値はパラメータとして受け取りハードコードしない。VPD 移動平均（SMA）線は指定時のみ凡例トグルで既定オフ（selected:false）で重ねる
  - 温湿度グラフの option ビルダー（既存）は一切変更しない（無回帰）
  - 観測可能完了: 生成 JSON に小文字 `yAxis` の markArea が3ゾーン含まれ、しきい値が反映され、SMA 指定時のみ凡例 selected:false・`</script>` 不混入を検証するユニットテストが緑
  - _Requirements: 4.1, 4.2, 4.3, 7.1, 7.2, 7.3_

## 4. Core — VPD パネルの View と handler

- [x] 4.1 VPD パネルの表示用 DTO 追加
  - VPD パネルの表示データ（option JSON・VPD 色・作物名/「既定」・適正帯上下限の表示文字列・VPD 数値カード・滞在率バー比率・時間帯別逸脱行）をイミュータブルな表示用構造体として定義し、グラフ領域 View に VPD パネルを内包させる
  - 観測可能完了: グラフ領域 View が VPD パネル DTO を保持し、ビルドが通る（既存温湿度フィールドは不変）
  - _Requirements: 4.3, 5.1, 6.1_

- [x] 4.2 VPD パネル組立（滞在率・時間帯別逸脱・カード・グラフ）
  - 生計測行＋デバイスの作物から、適正帯（未設定は既定）を解決し、VPD 系列・滞在率・期間移動平均（既存 SMA・期間別窓幅を共用）・VPD カード（現在/期間平均/滞在率/最大逸脱）・グラフ y 上限を算出し VPD パネル DTO を組む
  - 時間帯別逸脱は時刻バケット（JST の時間帯）ごとに平均 VPD・在帯率・主たる逸脱方向（乾き/湿り）を集計する（時刻を要する集計は handler 境界で・純粋層に時刻を持ち込まない）
  - 滞在率は瞬間値・平均でなく「計測のうち適正帯に入っていた時間割合」として算出する
  - 観測可能完了: 在帯率・最大逸脱・時間帯別バケット・作物別しきい値適用を、計測行入力で検証するユニットテストが緑
  - _Requirements: 4.2, 4.3, 5.1, 5.2, 5.3, 6.1, 6.2, 7.2_
  - _Depends: 1.1, 2.1, 2.2_

- [x] 4.3 グラフ領域への VPD パネル統合（Show/Chart の作物受け渡し・無回帰）
  - グラフ領域組立にデバイス（作物）を渡すよう変更し、データ有時に VPD パネルを組み込む。デバイス詳細表示・期間切替フラグメントの双方が所有者認可で取得済みのデバイスを渡す
  - 期間切替（24h/3d/7d/30d）で温湿度グラフと VPD パネルの双方が当該期間で更新される。計測 0 件時は VPD パネルを出さずデータ未到着を破綻なく示す
  - 温湿度グラフ・統計カード・日次表の出力は不変（無回帰）。所有者認可は既存を消費し他者要求は拒否される
  - 観測可能完了: 期間切替で VPD パネルを含むグラフ領域フラグメントが返り、空データで非表示になり、温湿度 option 文字列が従来と一致するハンドラテストが緑
  - _Requirements: 4.3, 4.4, 4.5, 5.3, 5.4, 6.2, 6.3, 8.1, 8.2, 8.5, 9.1, 9.2_
  - _Depends: 4.2_

- [x] 4.4 VPD パネルの templ 描画
  - グラフ領域テンプレ（データ有・日次表の下）に VPD パネルを描画する: VPD チャートのマウント先（`kPa` 単位指定）と option script、VPD 数値カード、滞在率バー（比率は動的幅）、時間帯別逸脱表。器はモックを写経し独自クラスは滞在率バーのみ
  - VPD パネルに作物名（未設定は「既定」）と適正帯上下限を表示する。VPD 移動平均は凡例トグルで既定オフ
  - 研究用表示に限定し、農家向けの良否ラベル・信号色・圃場共有 URL・アラート連動は含めない
  - `make templ` で生成し、既存クライアント初期化（VPD 線は既存 `data-echarts` 経路で描画・改修なし）で描画される
  - 観測可能完了: 詳細画面の描画 HTML に VPD チャート要素（kPa 単位）・作物名/適正帯・滞在率バー・逸脱表が含まれ、農家向け表示が含まれないことを検証する templ レンダリングテストが緑
  - _Requirements: 4.1, 4.2, 4.3, 5.1, 6.1, 7.1, 7.3, 9.3, 9.4_
  - _Depends: 3.1, 4.1, 4.3_

## 6. Validation — 統合・無回帰

- [x] 6.1 詳細画面/期間切替の統合・無回帰テスト

  - 観測可能完了: VPD 追加後も温湿度可視化・認可が従来同等で、VPD パネル描画と空データ分岐が緑の統合テスト群

  - 観測可能完了: 作物の描画・不正→422・保存・CSRF 往復を検証する統合テストが緑
